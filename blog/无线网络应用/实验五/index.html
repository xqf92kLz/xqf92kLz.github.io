    实验五：无线网络 DHCP 服务及 AP 组网

    2025-12-182025-12-18

    第六次课

    • 实验整体概述

      • 本实验分为两大核心模块（无线网络 DHCP 服务配置应用实验、无线 AP 组网实验）及两个补充选做实验，旨在帮助学生掌握 DHCP 服务的配置与故障转移技术、无线 AP 的多种工作模式及组网方法，巩固无线网络设备的配置与应用能力。实验涉及 TL-CPE210 型无线 AP、TL-XDR1860 型无线路由器及 Windows Server 操作系统，需按组完成设备领取与配置，核心要求为设备间能动态获取 IP 地址并正常通信。

      • 通用设备连接注意事项

        • 无线 AP 的指示灯、RESET 键、POE 口和 LAN 口均在设备侧顶部；
        • POE 口通过黑色 PoE 适配器连接网线，兼具“供电+传输数据”功能，请勿随意拔出；
        • 配置 AP 时需使用 Chrome 或 Firefox 浏览器（不支持 IE），默认登录 IP 为 192.168.1.254，用户名/密码通常为 admin/admin123（以设备标注为准）。

    • 第一部分：无线网络 DHCP 服务配置应用实验

      • 实验目的
        • 了解 DHCP 服务的工作原理和流程；
        • 掌握 TL-CPE210 无线 AP/TL-XDR1860 无线路由器的 DHCP 配置技术；
        • 掌握 Windows Server 的 DHCP 服务配置及故障转移技术；
        • 巩固无线网络设备的互联与通信测试方法。

      • 实验原理：DHCP 简介

        • 定义：DHCP（动态主机配置协议）是简化 IP 配置的 TCP/IP 标准，通过服务器集中管理 IP 地址池，为客户端动态分配 IP 地址、子网掩码、网关等参数，减少手工配置成本，提高 IP 资源利用率。
        • 核心概念：
          • 作用域：对子网中 DHCP 客户端的 IP 管理分组，包含 IP 地址范围、子网掩码、租约期限等属性；
          • 租约：客户端使用动态 IP 的有效时间，到期可续租；
          • 服务器端：集中管理 IP 地址池和配置参数；客户端：自动获取服务器分配的 IP 配置。
        • 管理工具：Windows 下通过“控制面板→管理工具→DHCP”控制台配置，核心命令：
          • ipconfig：查看 IP 地址；
          • ipconfig /all：查看详细 IP 配置（含 DHCP 服务器地址）；
          • ipconfig /release：释放已获取的 IP 地址；
          • ipconfig /renew：重新申请 IP 地址。

      • 实验 1.1：配置无线 AP 为 DHCP 服务器

        • 实验拓扑图

          PC1（动态获取 IP） ←→ 无线 AP（192.168.2.1，DHCP 启用） ←→ PC2（动态获取 IP）
                                            ↑
                                            ↓
                                          PC3（动态获取 IP）

        • 实验步骤（简明+详细补充）

          • 复位无线 AP：恢复出厂设置，默认 IP 为 192.168.1.254；
          • AP 有线连接与配置页面访问：用 1 台 PC 通过有线网卡连接 AP 的 LAN 口，将 PC 有线网卡 IP 设为 192.168.1.*（如 192.168.1.4），打开浏览器输入 192.168.1.254，登录 AP 配置页面；
          • 配置 AP 无线参数：
            • 进入“无线设置→基本设置”，设置 SSID（自定义，如 TL-AP-DHCP）；
            • 信道/频率按“大组通道号分配表”设置（如大组一设为 2）；
            • 加密方式设为“无加密”（实验环境），开启 SSID 广播；
            • 确定后重新登录 AP，再次保存配置（确保生效）；
          • 配置 AP 网络参数与 DHCP 服务：
            • 进入“网络参数→LAN 设置”，将 AP 的 IP 改为 192.168.2.1（子网掩码 255.255.255.0）；
            • 启用“DHCP 服务器”，设置地址池范围：192.168.2.100~192.168.2.199（x>1、y>x 即可）；
            • 缺省网关设为 192.168.2.1，地址租期设为 120 分钟；
            • 确定后保存，重新登录 AP 确认配置；
          • DHCP 客户端配置：
            • 禁用 3 台 PC 的有线网卡，启用无线网卡；
            • 所有 PC 的 TCP/IP 设置改为“自动获得 IP 地址”“自动获得 DNS 服务器地址”；
            • 确保 PC 的“Windows 管理工具→服务→DHCP Client”已启动；
          • 客户端连接与测试：
            • 3 台 PC 均无线连接到上述 AP 的 SSID；
            • 每台 PC 执行ipconfig /release→ipconfig /renew→ipconfig /all，查看是否获取 192.168.2.* 网段 IP，且“DHCP 服务器”地址为 192.168.2.1；
            • 3 台 PC 之间互相执行ping命令（如 PC1 ping PC2 的 IP），需全部连通。
        • 实验结果要求
          • PC1、PC2、PC3 均能从 AP 动态获取 192.168.2.* 网段 IP；
          • 三台 PC 之间 ping 测试无丢包，通信正常。

      • 实验 1.2：配置 PC 机为 DHCP 服务器（含故障转移）

        • 实验拓扑图

          PC1（动态获取 IP） ←→ 无线 AP（192.168.2.1，DHCP 禁用） ←→ PC2（辅 DHCP，192.168.2.2）
                                            ↑
                                            ↓
                                          PC3（主 DHCP，192.168.2.3）

        • 实验前提

          • 完成实验 1.1 的基础配置，需修改 AP 的 DHCP 设置（禁用）。

        • 实验步骤（含故障转移配置）

          • 阶段 1：基础配置（AP+PC 静态 IP）
            • 修改 AP 配置：
              • 登录 AP 配置页面，进入“网络参数→LAN 设置”，禁用 DHCP 服务器；
              • 保存配置，确保 AP 的 IP 仍为 192.168.2.1；
            • PC3（主 DHCP 服务器）静态 IP 设置：
              • 禁用 PC3 的有线网卡，启用无线网卡，连接 AP；
              • TCP/IP 设置改为“使用下面的 IP 地址”：IP=192.168.2.3，子网掩码=255.255.255.0，网关=192.168.2.1；
            • PC2（辅 DHCP 服务器）静态 IP 设置：
              • 操作同 PC3，IP 设为 192.168.2.2（暂不连接 AP）；
            • PC1（客户端）配置：保持“自动获得 IP 地址”，暂不连接 AP。
          • 阶段 2：主 DHCP 服务器（PC3）配置
            • 打开 PC3 的“Windows 管理工具→DHCP”控制台；
            • 连接 DHCP 服务器：
              • 若未自动连接，右键“DHCP”→“添加服务器”，输入 PC3 的 IP（192.168.2.3）；
            • 新建作用域：
              • 展开 PC3 的“IPv4”，右键→“新建作用域”，启动向导；
              • 作用域名称：自定义（如 DHCP-Test），描述可选；
              • IP 地址范围：192.168.2.100~192.168.2.199，子网掩码=255.255.255.0；
              • 添加排除地址（可选）：如 192.168.2.150~160（不分配给客户端）；
              • 租约期限：8 天（可自定义）；
              • 配置 DHCP 选项：选择“是，我想现在配置这些选项”；
              • 路由器（默认网关）：输入 192.168.2.1，添加；
              • DNS 服务器：无需填写（实验无需外网）；
              • 激活作用域：选择“是，我想现在激活此作用域”；
            • 验证配置：查看 DHCP 控制台的“地址池”，确认 IP 范围、排除地址无误。
          • 阶段 3：客户端（PC1）测试与 PC2 接入
            • PC3 先无线连接 AP，再将 PC1 无线连接 AP；
            • PC1 执行ipconfig /release→ipconfig /renew→ipconfig /all，确认“DHCP 服务器”地址为 192.168.2.3（从主 DHCP 获取 IP）；
            • 将 PC2 无线连接 AP，确保 PC2 的“DHCP Server”服务已启动；
            • 三台 PC 之间互相 ping 测试，需全部连通。
          • 阶段 4：DHCP 故障转移配置
            • 前提条件：
              • PC3 与 PC2 之间能互相 ping 通（连通性验证）；
              • PC3 与 PC2 的系统时间差≤60 秒（时间同步）；
              • PC2 的 DHCP 控制台“IPv4”无作用域（如有则删除）；
            • 配置故障转移：
              • 在 PC3 的 DHCP 控制台，右键“作用域[192.168.2.0]”→“配置故障转移”；
              • 选择可用作用域：192.168.2.0，点击“下一步”；
              • 伙伴服务器：输入 PC2 的 IP（192.168.2.2），点击“下一步”；
              • 新建故障转移关系：
                • 关系名称：自定义（如 PC3-PC2-Failover）；
                • 模式：热备用服务器；
                • 伙伴服务器角色：活动（PC3 为活动服务器，PC2 为备用）；
                • 不选中“启用消息验证”；
                • 点击“下一步”→“完成”；
            • 验证故障转移关系：
              • 分别查看 PC3 和 PC2 的 DHCP 控制台“IPv4”，均能看到故障转移关系（若未显示，按 F5 刷新）。
          • 阶段 5：故障转移测试
            • 模拟主 DHCP（PC3）故障：
              • 在 PC3 的 DHCP 控制台，右键 PC3 的服务器名称→“所有任务→停止”（或断开 PC3 的网络）；
            • PC1 执行ipconfig /release→ipconfig /renew→ipconfig /all；
            • 验证结果：PC1 的“DHCP 服务器”地址变为 192.168.2.2（从辅 DHCP 获取 IP），且 IP 地址与之前从 PC3 获取的一致，说明故障转移生效。
        • 实验结果要求
          • PC1 能从主 DHCP（PC3）或辅 DHCP（PC2）动态获取 IP；
          • 三台 PC 之间互相 ping 通；
          • 主 DHCP 故障时，辅 DHCP 能自动接管服务，PC1 正常获取 IP。

    • 第二部分：无线 AP 组网实验

      • 实验目的
        • 了解无线 AP 的 6 种工作模式（AP、Bridge、Repeater、Client、AP Router、AP Client Router）；
        • 掌握 TL-CPE210 无线 AP 的 Bridge 模式配置技术；
        • 理解无线 AP 组网的“同一网段”要求，巩固设备互联测试方法。

      • 实验原理：无线 AP 工作模式简介

        工作模式 核心功能 实验应用场景
        AP 模式（默认） 构建集中控制式网络，转发所有通信，类似有线交换机，支持无线/有线客户端接入 组网实验中作为核心节点
        Bridge 模式 点到点桥接，相当于无线“网线”，连接两个小网络，需与 AP 模式配合使用 实验 2.1（必做）
        Client 模式 等同于无线网卡，无自身 SSID，仅支持有线客户端接入，从属于 AP 模式设备 实验 2.2（仅供参考）
        Repeater 模式 中继放大信号，延伸覆盖范围，无自身 SSID，继承前端 AP 的 SSID 补充实验场景
        AP Router 模式 类似家用路由器，前端有线接入外网，无线覆盖范围更广 补充选做实验一
        AP Client Router 模式 无线接入前端网络（如 ZJUWLAN），为后端客户端提供无线/有线接入 补充选做实验四

      • 实验 2.1：Bridge 模式的无线 AP 组网实验（必做，需写实验总结）

        • 实验核心要求
          • 1 台 AP（AP2）设为 AP 模式，1 台 AP（AP1）设为 Bridge 模式；
          • 两 AP 的信道、安全模式相同，SSID、IP 地址不同；
          • 三台 PC 均设为静态 IP，属于同一网段（192.168.1.*），互相 ping 通。

        • 实验拓扑图

          PC1（192.168.1.20） ←→ AP1（192.168.1.5，Bridge 模式） ←无线→ AP2（192.168.1.10，AP 模式） ←→ PC2（192.168.1.6）
                                                                                                  ↑
                                                                                                  ↓
                                                                                                PC3（192.168.1.7）

        • 实验步骤

          • 阶段 1：AP 复位与基础配置（AP1 和 AP2）
            • 复位 AP1 和 AP2，默认 IP 均为 192.168.1.254；
            • 配置 AP2（AP 模式）：
              • 用 PC2（或 PC3）通过有线网卡连接 AP2 的 LAN 口，PC2 有线 IP 设为 192.168.1.8；
              • 登录 AP2 配置页面（192.168.1.254）；
              • 网络参数：修改 AP2 的 IP 为 192.168.1.10（子网掩码 255.255.255.0），禁用 DHCP 服务器；
              • 工作模式：确认右上角为“Access Point”模式；
              • 无线参数：
                • SSID：自定义（如 TL-AP2），开启 SSID 广播；
                • 信道：按大组分配表设置（如大组一设为 2）；
                • 安全模式：无加密；
              • 确定后保存，重新登录 AP2 确认配置；
            • 配置 AP1（Bridge 模式）：
              • 用 PC1 通过有线网卡连接 AP1 的 LAN 口，PC1 有线 IP 设为 192.168.1.4；
              • 登录 AP1 配置页面（192.168.1.254）；
              • 网络参数：修改 AP1 的 IP 为 192.168.1.5（子网掩码 255.255.255.0），禁用 DHCP 服务器；
              • 工作模式：右上角改为“Bridge”模式，保存后重新登录；
              • 无线参数：
                • AP 设置：SSID 自定义（如 TL-AP1，与 AP2 不同），开启 SSID 广播，无线射频启用；
                • Client 设置：点击“扫描”，在弹出列表中选中 AP2 的 SSID（如 TL-AP2），点击“连接”（确保加密方式与 AP2 一致）；
              • 确定后保存，重新登录 AP1 确认配置。
          • 阶段 2：PC 静态 IP 设置与连接
            • 三台 PC 均禁用有线网卡，启用无线网卡；
            • 设置静态 IP（子网掩码均为 255.255.255.0，网关为空或设为所连 AP 的 IP）：
              • PC1：192.168.1.20；
              • PC2：192.168.1.6；
              • PC3：192.168.1.7；
            • PC1 无线连接 AP1 的 SSID（TL-AP1），PC2、PC3 无线连接 AP2 的 SSID（TL-AP2）。
          • 阶段 3：组网测试与状态验证
            • 三台 PC 之间互相 ping 测试，需全部连通；
            • 验证 AP 状态：
              • AP1 的“系统状态”页：同时显示 AP1 和 AP2 的信息（射频状态、前端 AP 的 SSID/BSSID）；
              • AP2 的“系统状态”页：仅显示 AP2 自身的信息；
              • 查看 AP1 和 AP2 的“无线客户端”信息，对比差异（AP1 可看到前端 AP 的客户端，AP2 仅看到自身客户端）。
        • 关键注意事项
          • 两 AP 的信道必须相同（AP1 的信道从属于 AP2，扫描连接后自动同步）；
          • 两 AP 的安全模式必须相同（实验环境设为无加密）；
          • 所有设备的 IP 必须属于同一网段（192.168.1.*），否则无法互联（AP 相当于无线交换机，仅转发同一网段数据）。

      • 实验 2.2：Client 模式的无线 AP 组网实验（仅供参考，无需完成）

        • 核心特点
          • AP1 设为 Client 模式（无自身 SSID 和信道，从属于 AP2）；
          • AP1 仅支持有线客户端接入（PC1 需有线连接 AP1）；
          • 所有设备 IP 仍为 192.168.1.* 网段，三台 PC 互相 ping 通。
        • 简化步骤
          • AP2 配置：同实验 2.1（AP 模式，IP=192.168.1.10，SSID=TL-AP2）；
          • AP1 配置：
            • 网络参数：IP=192.168.1.20（与 AP2 不同）；
            • 工作模式：改为“Client”模式；
            • 无线设置：扫描并连接 AP2 的 SSID，启用无线射频；
          • PC1 有线连接 AP1，PC2、PC3 无线连接 AP2；
          • 三台 PC 互相 ping 测试，需全部连通。

    • 补充选做实验

      • 实验三：无线路由器的无线互联实验（TL-XDR1860）
        • 实验要求
          • 自学 TL-XDR1860 无线路由器的 Web 配置页面及用户手册；
          • 设计拓扑图，用两种方法实现两台路由器及下属 PC 的无线互联；
          • 核心验证：路由器互联成功页面、PC 之间互相通信。
        • 实现方法
          • 方法 1：无线桥接：
            • 一台路由器设为 AP 模式（主路由器），另一台设为 Bridge 模式（从路由器）；
            • 确保两路由器的 SSID、信道、安全模式一致，IP 属于同一网段；
          • 方法 2：易展功能：
            • 访问主路由器 Web 页面，开启易展功能；
            • 按下从路由器的易展按钮，自动与主路由器配对；
            • 建议采用“网页配置+按钮配对”方式，避免多组实验干扰。
        • 思考问题
          • 无线互联的是路由器的 LAN 口还是 WAN 口？（答案：LAN 口，确保同一网段互联）
      • 实验四：基于无线 AP 的无线中继共享上网实验（TL-CPE210）
        • 实验要求
          • 自学 TL-CPE210 的 AP Client Router 模式配置；
          • 实现功能：中继 ZJUWLAN 信号，2 台无线客户端共用同一 VPN 账号上外网；
          • 核心验证：客户端无需重复登录 VPN，即可访问外网。
        • 关键提示
          • AP 地址末位需改为非 254（如 192.168.1.100）；
          • 两种实现方法对比：
            • 方法 1：无线 WAN 口中继（AP Client Router 模式，无线接入 ZJUWLAN）；
            • 方法 2：有线 WAN 口中继（需有线连接 ZJUWLAN 接入点）；
          • 思考问题：L2TP 服务器地址是否能用域名？（答案：支持，需确保 DNS 解析正常）

    • 思考题与实验检查要求

      • 思考题
        • MAC 地址有几位？（答案：48 位，12 个十六进制数）；
        • 获得 IP 地址的客户机的 MAC 地址在哪里查看？（答案：在 DHCP 服务器的控制台→“地址租用”界面，“唯一 ID”列即为 MAC 地址）。
      • 实验检查内容
        • 必查实验：实验 1.2（PC 机 DHCP+故障转移）、实验 2.1（Bridge 模式组网）；
        • 检查要点：
          • 实验 1.2：PC1 的 DHCP 服务器地址（主/辅切换）、故障转移关系页面、三台 PC ping 通；
          • 实验 2.1：两台 AP 的 Web 配置页面（无线设置页面）、三台 PC ping 通；
          • 补充实验：互联成功页面（桥接/易展、中继共享上网）。
    • 通用注意事项
      • 配置后必须“保存”并“重新登录”设备，确保配置生效；
      • 实验过程中如需修改 IP，需先释放原有 IP（ipconfig /release）；
      • 禁用无关网卡（如 DHCP 实验中禁用有线网卡，仅用无线网卡）；
      • 若设备配置异常，可通过 RESET 键复位（恢复出厂设置）；
      • 实验总结需重点撰写实验 1.1 和 2.1 的原理、步骤、结果及问题分析。
