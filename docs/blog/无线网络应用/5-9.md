第二次课

- 模块 5 数制系统
	- 核心内容：二进制、十进制、十六进制的转换
	- 转换工具：Windows“计算器（程序员模式）”，支持直接输入某进制数字，切换后获取对应进制结果
	- 应用场景：IP 地址（十进制表示）、MAC 地址（十六进制表示）、网络数据传输（二进制比特流）
- 模块 6 数据链路
	- MAC 地址（物理地址）
		- 定义：网络接口（网卡）的唯一标识，由 48 位二进制组成，表现为 12 个十六进制数字（如 AA:AA:AA:AA:AA:AA）
		- 特性：物理固化在网卡中，不随网络位置变化，类比 “人名”
		- 关键类型：广播 MAC 地址（FF-FF-FF-FF-FF-FF），用于向同一网络所有主机发送消息
	- 网络层次设计（解决 MAC 地址局限性）
		- 局限性：MAC 地址无层次，无法标识网络位置，大规模网络中效率低、广播风暴风险高
		- 层次结构：核心层（高速数据传输）→分布层（网络互联、流量控制）→接入层（主机接入），类比 “国家→省→市→街道”
	- IP 地址（逻辑地址）
		- 定义：标识主机的网络位置，分为 IPv4（32 位，十进制四段）和 IPv6（128 位）
		- 结构：网络部分（标识所在网络）+ 主机部分（标识网络内主机），有层次，类比 “住址”
		- 关键命令：ipconfig /all（查看主机 MAC 地址、IP 地址、默认网关等信息）
	- 网络测试命令
		- ping 命令：测试源和目的设备之间的端到端连通性，Ctrl+C 中止
		- tracert 命令（Windows）/traceroute（其他系统）：追踪路由路径，显示每一跳路由器及耗时，用于定位网络故障
- 模块 7 以太网交换
	- 交换机的核心功能
		- 端口连接：多端口设备，连接同一网络内的多台主机
		- MAC 地址表管理：自动学习接入主机的 MAC 地址与对应端口，动态更新表项
		- 帧转发逻辑：
			- 已知目的 MAC：在源端口与目的端口间创建临时专用通道，避免冲突
			- 未知目的 MAC：采用 “泛洪” 方式，转发至所有端口（除源端口），主机自行匹配 MAC 地址
		- 广播处理：转发广播帧至所有端口，交换机连接的所有主机属于同一广播域（延伸广播域）
	- 冲突避免机制：以太网使用 CSMA/CD（有线），WLAN 使用 CSMA/CA（无线），解决多设备同时发送数据的冲突问题
- 模块 8 网络层
	- 路由器的本质与组成
		- 本质：一台专用计算机，运行 Cisco IOS 操作系统
		- 核心组件：CPU（执行路由功能）、内存（RAM/ROM/NVRAM/ 闪存）、LAN/WAN 接口（连接不同网络）
	- 路由器的核心功能
		- 路由表管理：存储路由信息，每条路由含 “目的网络地址 + 出口接口 + 路由度量（跳数等）”
		- 数据包转发：解封装数据帧→读取 IP 地址的网络部分→查询路由表→封装新帧（更新 MAC 地址）→从对应接口转发
		- 广播域分割：不转发广播帧，每个路由器接口对应一个独立广播域，减少广播风暴
		- 关键命令：route print /netstat -r（查看路由表）
	- 默认网关的作用
		- 定义：本地网络的出口，即主机所在网络连接的路由器接口 IP 地址
		- 必要性：主机需配置默认网关才能与不同网络的主机通信，否则无法转发跨网络数据
	- 路由器的表格：同时维护路由表（网络级信息）和 ARP 表（本地网络主机 IP-MAC 映射）
	- 路由表条目配置方式：静态设定（管理员手动输入）、动态设定（从其他路由器收集信息更新）
	- 默认路由：目的网络不在路由表时，转发数据包的 “最后接口”，避免数据包被丢弃
- 模块 9 地址解析
	- 核心作用：实现同一网络内 “IP 地址→MAC 地址” 的映射，解决 “知 IP 不知 MAC” 的通信障碍
	- ARP 工作流程（发现与存储）
		- 步骤 1：源主机发送广播帧（目的 MAC=FF-FF-FF-FF-FF-FF），包含目的主机 IP 地址
		- 步骤 2：同一网络所有主机接收广播，仅 IP 地址匹配的主机响应，返回自身 MAC 地址
		- 步骤 3：源主机接收响应，将 “目的 IP + 目的 MAC” 存入 ARP 表，后续通信直接使用
	- 关键命令：arp -a（查看主机 ARP 表，含 IP-MAC 映射关系）
	- 代理 ARP：当主机访问远程网络时，发送 ARP 请求默认网关 IP，路由器接口响应自身 MAC 地址，替代目的主机响应

- 核心知识点总结
	- 地址核心：MAC 地址（物理标识，本地有效）+ IP 地址（逻辑位置，跨网络有效），二者缺一不可，分别对应数据帧和数据包的报头
	- 设备核心：交换机（二层设备，基于 MAC 转发，延伸广播域，同一网络通信）+ 路由器（三层设备，基于 IP 转发，分割广播域，跨网络通信）
	- 协议核心：ARP（IP→MAC 映射）、以太网协议（本地网络通信）、IP 协议（跨网络路由）、TCP/UDP（端到端传输）
	- 通信规律：跨网络传输时，IP 地址不变（标识最终收发方），MAC 地址逐跳更新（适配当前链路）
	- 关键流程：数据封装（自上而下加报头）→传输（交换机 / 路由器转发）→解封装（自下而上剥报头）→数据交付
	- 核心命令：ipconfig /all（查地址信息）、arp -a（查 ARP 表）、ping（连通性测试）、tracert（路由追踪）、route print（查路由表）
